Playwright Test Automation Suite on TestMU AI

E2E Testing & Cloud Integration

A robust and resilient E2E (End-to-End) test automation suite built with Playwright designed to run seamlessly on the TestMU AI (LambdaTest) cloud grid.

This repository contains robust test scripts that automate user interaction scenarios on the LambdaTest Selenium Playground.

📁 Repository Structure

  • tests/: Contains the Playwright test specification files:
    • scenario1.spec.js: Automates the Simple Form Demo scenario.
    • scenario2.spec.js: Automates the Drag and Drop Sliders scenario.
    • scenario3.spec.js: Automates the Input Form Submit validation scenario.
  • playwright.config.js: Global configuration file defining environment settings, local browser options, and the integration connection to the LambdaTest remote grid.
  • SimpleFormDemo-test-results/: Execution results, logs, and screenshots for Scenario 1.
  • DragAndDropSliders-test-results/: Execution results, logs, and screenshots for Scenario 2.
  • InputFormSubmit-test-results/: Execution results, logs, and screenshots for Scenario 3.
  • .gitignore: Excludes locally generated folders like node_modules and runtime credentials.

🛠️ Test Scenarios

1. Simple Form Demo (scenario1.spec.js)

  • Navigates to the LambdaTest Selenium Playground.
  • Enters the message "Welcome to test MU AI".
  • Clicks Get Checked Value and asserts that the displayed paragraph matches the input text exactly.

2. Drag & Drop Sliders (scenario2.spec.js)

  • Navigates to the slider playground.
  • Locates the range slider for Default value 15 (under #slider3).
  • Sets the value to 95 using a native event dispatcher to ensure flawless UI state update.
  • Verifies that the adjacent output label displays 95.

3. Input Form Submit (scenario3.spec.js)

  • Navigates to the Input Form page.
  • Fills in all required fields (Name, Email, Password, Company, Country, City, Address, Zip, etc.).
  • Submits the form and verifies that the success message: "Thanks for contacting us, we will get back to you shortly." is visible on the page.

🚀 Getting Started

1. Install Dependencies

npm install

2. Configure Environment Variables

You must set your TestMU AI (LambdaTest) credentials in your terminal session before execution.

Windows (PowerShell):

$env:LT_USERNAME="your_username"
$env:LT_ACCESS_KEY="your_access_key"

macOS / Linux:

export LT_USERNAME="your_username"
export LT_ACCESS_KEY="your_access_key"

🏃 Running the Tests

Option A: Run Locally (Headless mode)

npx playwright test

Option B: Run on the TestMU AI Cloud Grid (Remote execution)

npx playwright test --project=lambdatest-chrome

Once triggered, the test runs on the LambdaTest cloud grid and all logs, console output, screenshots, and video recordings will be accessible directly on the TestMU AI Automation Dashboard.
